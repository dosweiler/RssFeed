      UniFi LTE 6.2.52

      Source: UniFi Access Point/Switch/LTE | Releases | Ubiquiti Community | Date: 2023-01-16 |

      Improvements [U-LTE] Fix incorrect APN parameters in some cases. Additional information Instead of using the manual links, you can also subscribe to your desired firmware channel, as described here. All URLs use HTTPS. This assumes your device(s) are running 3.7.56 or later and that the device's date is properly set. Please change URLs to HTTP for device(s) running earlier firmware or if the date is incorrect (e.g. if device can't reach a NTP server).   All feedback is welcome, including lettings us know about successful device upgrades. Please ensure that you include the following information at a minimum: Network Application version. Firmware version (previous and current).  Device model(s).

  • 2022-12-21

      UniFi Switch 6.4.18

      Source: UniFi Access Point/Switch/LTE | Releases | Ubiquiti Community | Date: 2022-12-21 |

      Improvements [USW] Add PoE fault event in syslog. [USW] Improve device inform scheduling. [USW] Improve inform URL resolving for IPv6 domain name. [USP-RPS] improve the RPS power activate detection. [USW-Flex] Enable PoE-injector mode by default. [USW-Pro-Aggregation] Enlarge message queue and improve the critical section handling. Bugfixes [USW] Fix inform URL override with default URL in a rare case. [USW-PoE] Fix can't unblock a port after disable port-security. [USW-PoE] Fix incorrect port speed if LAG member port not used. [USW-PoE] Fix missing TX/RX error counter reading. [USW-Pro] Fix incorrect RPS Activated screen when no RPS connected. [USW-Pro] Fix DHCP snooping issue with Logitech Harmony Hub in 6.3.14. [USW-Flex] Fix PoE output budget is only 5W if powered via PoE+. [USW-Flex] Fix AP getting into low performance mode when connected to PoE+ port. [USW-Lite-8-PoE] Fix PDs do not turn back on when you choose "Cycle PoE" during switch restart. [USW-Enterprise-8-PoE] Fix device offline after update native VLAN in switch port profile of uplink port. Additional information Please report any issues you encounter with this release using the Report Bug form. Instead of using the manual links, you can also configure the Device Firmware Release Channel in the Network Application (enable Early Access Release Channels) Your device built-in display may turn black or unresponsive for ~10 minutes after firmware upgrades to upgrade LCM firmware. All URLs use HTTPS. This assumes your device(s) are running 3.7.56 or later and that the device's date is properly set. Please change URLs to HTTP for device(s) running earlier firmware or if the date is incorrect (e.g. if device can't reach a NTP server).   Any and all feedback is welcome, including lettings us know about successful device upgrades. When reporting an issue, please ensure that you include the following information: Network Application version. Firmware version (previous and current).  Device model(s).
